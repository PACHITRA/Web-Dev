Introduction

This tutorial explores the benefits of using semantic HTML elements instead of div tags. Semantic elements improve the structure, SEO, and accessibility of web pages.

Key Points Covered

1. Semantic HTML Elements Are Preferred Over Divs

  • Semantic elements provide clear meaning and structure to HTML, making it easier for developers and users to scan the code.
  • They improve Search Engine Optimization (SEO), helping search engines rank websites higher.
  • Enhanced accessibility for screen readers and better content categorization/indexing.

2. Semantic Structure of a Typical Web Page

  • The header often contains a logo and navigation links.
  • The main content is divided into section elements with headings and paragraphs.
  • Example: The footer includes links and nested structures for additional content.

3. Using Tags Like Sidebar, Article, and Aside

  • Sidebars are typically marked with aside, separating them from the main content.
  • The article tag represents self-contained content, making it flexible to move to another location without losing meaning.
  • The nav tag is reserved for primary navigation links.

4. Replacing Divs with Sections

  • The section tag groups related content for semantic clarity.
  • Div tags are used only for layout purposes or unrelated content.
  • Example: A section grouping company logos maintains semantic meaning compared to using a div.

5. Differentiating Between Sections and Divs

  • Sections denote related content as part of a larger context.
  • Divs are for layout or wrapping content that lacks semantic meaning.
  • Proper use of sections and divs improves a webpage’s structure and accessibility.

6. Organizing Content with Aside and Article Tags

  • The aside tag is used for content like sidebars that are secondary to the main page.
  • The article tag is used for main content that stands on its own, offering clarity and flexibility.

7. Using Tags for Navigation and Structuring

  • The nav tag is commonly used for organizing navigation menus.
  • Use div tags for purely stylistic elements or unrelated content.

8. Styling Without Overloading Semantics

  • Visually similar elements, like breadcrumbs and buttons, serve different purposes.
  • Breadcrumbs are typically marked with nav or ul, while buttons are marked with the button tag.
  • Using semantic tags ensures proper functionality while achieving the desired layout.

Conclusion

Semantic HTML elements make web pages more readable, accessible, and SEO-friendly. They allow developers to create meaningful structures that benefit both humans and search engines. Proper use of elements like header, section, aside, and article enhances the overall quality and usability of websites.

3. What is Focus?

The concept of "focus" is crucial for web accessibility. It indicates which element on a webpage is currently active and ready to receive keyboard input. Focusable elements include links, buttons, and form inputs.

4. Controlling Focus with tabindex

Use the tabindex attribute to control the focus order of elements when navigating with the Tab key. The tabindex="0" includes an element in the natural tab order, while tabindex="-1" removes it from the tab order but keeps it focusable programmatically.

Example: 
Link
Tip: Avoid using positive tabindex values, as they can confuse the tab order.

5. Just Use Button

Always use the <button> element for actions. Avoid using <div> or <span> for clickable actions as they are not semantically correct and might not be accessible for screen readers.
